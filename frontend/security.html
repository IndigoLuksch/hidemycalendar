How Security Works

  • A calendar subscription link is input
  • Using a secret key stored in Cloudflare's Secretes Store, the original URL is encrypted. A new, private URL of format .../?url= is generated. The original URL is not stored
  • When a calendar application requests event information from the private URL, a Cloudflare worker decreypts the encrypted URL, fetches event details, modifies them to show only as 'Busy', and passes this on to the calendar application.

    • Key security features:

    • the original URL is not stored by Cloudflare
    • event details are not stored by Cloudflare
    • the original URL is present in the private URL in encrypted form
