How Security Works

1. A calendar subscription link is input
2. Using a secret key stored in Cloudflare's Secretes Store, the original URL is encrypted. A new, private URL of format .../?url= is generated. The original URL is not stored
3. When a calendar application requests event information from the private URL, a Cloudflare worker decreypts the encrypted URL, fetches event details, modifies them to show only as 'Busy', and passes this on to the calendar application.
